BUILD REPORT — webOS-25 GStreamer demuxers with DTS re-enabled
================================================================
Date: 2026-07-22
Target: LG C5 (webOS 25), 32-bit ARM EABI5 soft-float (arm-linux-gnueabi),
        glibc 2.35, GStreamer 1.24

Build approach that succeeded: APPROACH 2 (LG source, full mini-stack)
----------------------------------------------------------------------
Built from LG's own webOS-25 monorepo source (GStreamer 1.24.0), i.e. all LG
customizations in qtdemux/tsdemux are preserved (DV-in-mp4, trickplay,
decproxy hints, webOS-specific caps fields). No Debian/upstream plugin source
was used — the "last-resort fallback" was NOT needed.

Approach 1 (link against Debian armel GStreamer 1.22 dev packages) was not
viable because Debian bullseye's armel repos no longer exist on
deb.debian.org (armel dropped from bullseye dists in Aug 2025) and bullseye
only ever shipped GStreamer 1.18 anyway. Instead, the minimal LG 1.24 stack
was cross-built inside debian:11-slim (glibc 2.31 build env → symbol ceiling
far below 2.35):
  1. gstreamer (core)                        -Dtools/tests/etc=disabled
  2. gst-plugins-base (libraries only)       -Dauto_features=disabled
  3. gst-plugins-good                        -Dauto_features=disabled -Disomp4=enabled -Ddca=true
  4. gst-plugins-bad                         -Dauto_features=disabled -Dmpegtsdemux=enabled -Ddca=true
All with meson 1.4.2 cross file (arm-linux-gnueabi-gcc 10.2, soft-float).
armel sysroot packages (libglib2.0-dev:armel 2.66.8, zlib1g-dev:armel) came
from snapshot.debian.org 20250601T000000Z pinned at priority 1001 (last
snapshot where bullseye/main still carried armel).

-Ddca=true sets #define DTS_SUPPORT in config.h of both plugin sets, so the
LG #ifdef DTS_SUPPORT code paths in qtdemux.c / tsdemux.c are compiled in.

One minimal build-system patch (no C code touched):
  gst-plugins-bad/gst-libs/gst/mpdclient/meson.build — LG bug: pkg_name /
  gstmpdclient are referenced after the "if xml2_dep.found()" endif, which
  aborts configuration when dash is disabled. The endif was moved to end of
  file (block skipped entirely; mpdclient is unrelated to tsdemux).

Notes / tradeoffs
-----------------
- Both .so were built against our own 1.24.0 core/base/bad libs; on the TV
  they resolve against LG's shipped 1.24 libs by soname (.so.0), same as the
  earlier dtsdec/truehd builds. libgstmpegtsdemux.so additionally NEEDS
  libgstcodecparsers-1.0.so.0 and libgstmpegts-1.0.so.0, both of which ship
  with LG's gst-plugins-bad on the device (LG's own tsdemux has the same
  deps).
- Binaries were stripped (--strip-unneeded) to match device convention;
  unstripped versions can be regenerated with build-demux.sh.
- Not tested on-device (per instructions: no TV connection). Static ABI
  checks all pass.

Reproduce: ./build-demux.sh <webos25-monorepo-path> <out-dir>
